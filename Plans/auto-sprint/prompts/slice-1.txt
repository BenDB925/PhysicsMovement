## Plan (approved — proceed without asking questions)

Replace the sprint button input with a timer-based auto-sprint system in `PlayerMovement.cs`.

### What to change

1. **Add new serialized fields** (after the existing sprint fields, around line 48-52):
   ```csharp
   [Header("Auto-Sprint")]
   [Tooltip("Seconds of sustained movement before sprint activates automatically.")]
   [SerializeField] private float _autoSprintDelay = 1.2f;

   [Tooltip("Seconds of zero move input before the auto-sprint timer resets.")]
   [SerializeField] private float _autoSprintStopGraceWindow = 0.1f;
   ```

2. **Add private state fields** (near the other sprint state fields):
   ```csharp
   private float _autoSprintTimer;
   private float _autoSprintStopGraceTimer;
   ```

3. **In FixedUpdate**, after move input is snapshotted and before sprint is consumed, add the timer logic:
   - If `_overrideSprintInput` is true: skip entirely (test seam preserved — `SetSprintInputForTest` still works)
   - Else if move input magnitude > 0.1f AND `_characterState.CurrentState` is not `CharacterStateType.Fallen` AND not `CharacterStateType.GettingUp`:
     - `_autoSprintStopGraceTimer = _autoSprintStopGraceWindow`
     - `_autoSprintTimer += Time.fixedDeltaTime`
   - Else if `_autoSprintStopGraceTimer > 0f`:
     - `_autoSprintStopGraceTimer -= Time.fixedDeltaTime`
   - Else:
     - `_autoSprintTimer = 0f`
   - `_sprintHeld = _autoSprintTimer >= _autoSprintDelay`

4. **In Update()**: Remove (or comment out with `// AUTO-SPRINT: button polling removed`) the line that polls the sprint button:
   ```csharp
   _sprintHeld = _inputActions.Player.Sprint.IsPressed();
   ```
   Leave the sprint button binding in `PlayerInputActions.inputactions` — that's removed in slice 3.

5. **Do NOT change** `SetSprintInputForTest`, `_overrideSprintInput`, `SprintNormalized`, or any downstream consumers.
