## Problem

Currently when landing from a sprint jump:
1. Horizontal damping fires: retains 75% per-frame over ~8 frames = ~10% of speed survives
2. Movement force is suppressed for 0.65s via `_jumpPostLandingGraceTimer`
3. `SprintNormalized` decays to 0 during airborne (CharacterState == Airborne gates sprint off)
   → This causes walk-animation snap mid-air AND walk-speed on landing

Result: sprint jump → land → visible speed loss + animation snap back to walk.

---

## Changes Required — `PlayerMovement.cs` only

### 1. New serialized fields (add near other jump/sprint fields)

```csharp
[SerializeField] private float _landIntoRunSprintThreshold = 0.9f;
[SerializeField] private float _sprintLandingGraceDuration = 0.15f;
```

### 2. New private state fields

```csharp
private bool _wasSprintingAtLaunch;
private float _sprintNormalizedAtLaunch;
```

### 3. Capture sprint state at wind-up entry

In `TickJumpWindUp()`, where `_jumpPhase` is set to `JumpPhase.WindUp` (around line 796):

```csharp
_wasSprintingAtLaunch = _sprintNormalized >= _landIntoRunSprintThreshold;
_sprintNormalizedAtLaunch = _sprintNormalized;
```

### 4. Freeze SprintNormalized during airborne sprint jumps

In `UpdateSprintNormalized()`, add at the top of the method (before computing targetSprintNormalized):

```csharp
// Freeze sprint blend during airborne if we launched at sprint speed.
// Prevents walk-animation snap mid-jump.
if (_recentJumpAirborne && _wasSprintingAtLaunch)
{
    _sprintNormalized = _sprintNormalizedAtLaunch;
    return;
}
```

### 5. Land-into-run on touchdown

In the `_jumpLandingDetected` block (around line 577, where `_jumpLandingDetected = true`):

```csharp
if (_wasSprintingAtLaunch)
{
    // Skip horizontal damping — momentum is preserved passively through inertia.
    _jumpLandingDampingTimer = 0f;

    // Shorten the movement force suppression window so the character can re-accelerate quickly.
    _jumpPostLandingGraceTimer = _sprintLandingGraceDuration;

    // Force sprint state directly — same pattern as SetSprintInputForTest.
    _sprintHeld = true;
    _autoSprintTimer = _autoSprintDelay; // Keep timer in sync so sprint persists naturally.
}
else
{
    _jumpPostLandingGraceTimer = _jumpPostLandingGraceDuration;
}
```

**Important:** Remove or guard the existing `_jumpPostLandingGraceTimer = _jumpPostLandingGraceDuration`
assignment that fires unconditionally — the if/else above replaces it.
