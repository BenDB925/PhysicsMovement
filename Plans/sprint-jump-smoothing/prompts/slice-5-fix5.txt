## Slice 5 Fix 5 — Sprint Gap Touchdown Detection (LandingFrame=-1)

### What's broken
`SprintGap_WithRunUp_ClearsGapAndLandsOnFarPlatform` fails:
- Character physically reaches 3.76m (gap is 3.6m) — it DOES clear the gap
- `LandingFrame = -1` — touchdown never registers
- `SprintTouchdownBudgetFrames = 300`

This is a TEST HARNESS problem, not a runtime bug. The character lands correctly.

### Root cause
`CaptureJumpGapOutcome()` (~line 408) uses this condition to detect touchdown:

```csharp
bool touchdownStateObserved = wasAirborne &&
    (_rig.BalanceController.IsGrounded ||
     _rig.CharacterState.CurrentState != CharacterStateType.Airborne);
bool groundedOnFarPlatform = touchdownStateObserved && IsTouchdownOnPlatform(_farPlatform, _rig.Hips.position);
```

The Opus fix in commit `c8f1a34` added `CharacterStateType.Airborne` to `ShouldSuppressLocomotion()`.
This changes how the character transitions through states on landing, so the old touchdown detection
conditions may no longer fire together at the right moment.

Specifically: when the character lands on the far platform edge at sprint speed, it may briefly be in
`Moving` or `Fallen` state while `IsGrounded` is still false (physics settling), and the hips may be
over the platform before `IsGrounded` latches.

### The fix
Relax the touchdown detection to also catch:
- Hips position is over the far platform (`IsTouchdownOnPlatform` returns true)  
- AND character was airborne at some point
- Regardless of whether `IsGrounded` has latched yet

The simplest approach: check `IsTouchdownOnPlatform` every frame after `wasAirborne = true`,
not just on frames where `touchdownStateObserved` is true. The platform bounds check is the
reliable signal — state/grounded flags are too brittle.

```csharp
// Replace the current touchdownStateObserved + groundedOnFarPlatform logic with:
if (wasAirborne && landingFrame < 0 && IsTouchdownOnPlatform(_farPlatform, _rig.Hips.position))
{
    landingFrame = frame;
    landedOnFarPlatform = true;
}
```

Check whether `IsTouchdownOnPlatform` already uses raycasts and hips height bounds — it does
(lines ~560-603). It's robust enough to use as the primary signal without the state guard.
