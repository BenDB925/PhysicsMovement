## Slice 5 Fix — Opus Brief: Reversal Clamp Root Cause

### The problem
`JumpAirControl_FullReverseInput_RetainsAtLeastSeventyPercentOfForwardTravel` fails:
- Zero-input forward travel (baseline): 0.816m
- Full reverse input (`Vector2.down`) forward travel: 0.002m
- Required: ≥ 70% of baseline (≥ 0.571m)

Three previous fix agents failed on this. All focused on `PlayerMovement.cs`. That was the wrong file.

### The actual root cause
`LocomotionDirector.ComputeTurnSeverity()` reads `CurrentDesiredInput.MoveWorldDirection`.
With `Vector2.down` held during a jump, `MoveWorldDirection` points **backward** (180° from travel).
`headingDelta = Vector3.Angle(backward, bodyForward) = 180°`
→ `headingSeverity = 1.0`, `turnSeverity = 1.0`

A `turnSeverity` of 1.0 propagates into lean calculations, COM stabilization, and body support
throughout `LocomotionDirector` — treating the airborne player as if they're doing a full U-turn.
This kills forward carry almost entirely, hence 0.002m.

**This is NOT a PlayerMovement problem.** The facing is already gated during jump airborne
(commits `ff873c8`, `a5d9eb5`). The locomotion director reads `CurrentDesiredInput` independently
and still computes max turn severity from the raw reverse input direction.

### The fix
Gate `LocomotionDirector.GetCoordinatorDesiredInput()` during `_playerMovement.IsRecentJumpAirborne`.
When the character is airborne mid-jump, pass forward-intent (zero move input, or the captured launch
direction) instead of the current reverse input to `TurnSeverity` and related downstream consumers.

The simplest correct approach:

```csharp
// In LocomotionDirector.GetCoordinatorDesiredInput() (line ~1134):
DesiredInput desiredInput = _playerMovement.CurrentDesiredInput;

if (_playerMovement.IsRecentJumpAirborne && !_playerMovement.IsGrounded)
{
    // During jump arc, pass zero move intent to the locomotion director
    // so turn-severity and lean calculations don't fight the preserved carry.
    // The airborne correction force in PlayerMovement handles any actual trim.
    return new DesiredInput(
        Vector2.zero,
        Vector3.zero,
        desiredInput.FacingDirection,
        desiredInput.JumpRequested,
        desiredInput.SprintNormalized);
}
```

**Check what `IsRecentJumpAirborne` and `IsGrounded` are exposed as on PlayerMovement** —
they may be `IsRecentJumpAirborne` (bool property) and `_balance.IsGrounded` may need to be
accessed via `BalanceController`. Look at the existing uses in `BalanceController.cs` lines
1448, 1493, 1749 for the correct property names.
