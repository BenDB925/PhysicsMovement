## Slice 5 Fix 4 — Fix JumpGapOutcomeTests crash in PrepareAirControlScenario

### What's broken
The test `JumpAirControl_FullReverseInput_RetainsAtLeastSeventyPercentOfForwardTravel` crashes
Unity's test runner before producing any results. The crash is in `PlayerPrefabTestRig.cs:42`
(`PlayModeSceneIsolation.ResetToEmptyScene()`).

Root cause: `PrepareAirControlScenario()` calls `_rig?.Dispose()` and then creates a brand new
`PlayerPrefabTestRig` inside a running coroutine. This resets the scene under the running test,
which crashes the test runner. You cannot dispose and recreate the rig mid-test.

### The fix
Rewrite `JumpAirControl_FullReverseInput_RetainsAtLeastSeventyPercentOfForwardTravel` so it:
1. Sets up the rig ONCE in normal SetUp (or at the start of the test)
2. Measures baseline (zero input) forward travel in the first run
3. Respawns/repositions the character on the SAME rig (without Dispose/recreate) for the second run
4. Measures reverse input forward travel in the second run
5. Computes ratio and asserts

To reposition without recreating the rig, use `RepositionRagdoll(_rig.RagdollSetup, _rig.HipsBody, spawnPosition)`
followed by a settle period. This is already used in other tests in this file.

The `PrepareAirControlScenario` helper must NOT call `_rig?.Dispose()` or `PlayerPrefabTestRig.Create()`.
If it needs to reset platforms, only destroy/recreate the platform GameObjects, not the rig.
