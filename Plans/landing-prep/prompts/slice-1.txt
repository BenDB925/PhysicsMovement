The problem: after a sprint jump, the character lands with a ~45° forward lean and takes ~0.5s to recover before locomotion resumes. Root cause (verified in code): `LegAnimator.OnCharacterStateChanged` calls `_jointDriver.SetSpringMultiplier(1f)` in a single frame — a 6.7× snap from 0.15x (airborne) to 1.0x (full stiffness). This creates a rigid-impact event where horizontal momentum pivots the torso forward over suddenly-rigid feet.

The exact line (LegAnimator.cs, `OnCharacterStateChanged` method, in the `else if (previousState == CharacterStateType.Airborne)` block):
```
_jointDriver.SetSpringMultiplier(1f);  // ← THIS IS THE SNAP — replace with ramp
```

## YOUR TASK: Slice 1 — Gradual Spring Restoration

### What to build

Replace the instant spring snap on landing with a time-based ramp.

**New serialized fields on LegAnimator:**
```csharp
[SerializeField, Tooltip("Seconds to ramp leg spring from airborne stiffness to full after landing. 0 = instant (legacy behaviour).")]
private float _landingSpringRampDuration = 0.12f;

[SerializeField, Tooltip("Exponent for the spring ramp curve. 0.5 = square-root ease (fast start), 1.0 = linear, 2.0 = slow start. 0.5 is recommended.")]
private float _landingSpringRampCurve = 0.5f;
```

**New private state on LegAnimator:**
```csharp
private float _landingSpringRampTimer;     // counts down from _landingSpringRampDuration
private float _landingSpringStartMultiplier; // captures airborne multiplier at touchdown
```

**In `OnCharacterStateChanged`, replace the instant snap:**

```csharp
// BEFORE:
_jointDriver.SetSpringMultiplier(1f);

// AFTER:
if (_landingSpringRampDuration > 0.0001f)
{
    _landingSpringRampTimer = _landingSpringRampDuration;
    _landingSpringStartMultiplier = _airborneSpringMultiplier;
    // Don't call SetSpringMultiplier here — FixedUpdate drives the ramp
}
else
{
    _jointDriver.SetSpringMultiplier(1f); // instant fallback
}
```

**In `FixedUpdate`, drive the ramp (add near the top of FixedUpdate, before other logic):**

```csharp
if (_landingSpringRampTimer > 0f)
{
    _landingSpringRampTimer -= Time.fixedDeltaTime;
    float t = 1f - Mathf.Clamp01(_landingSpringRampTimer / _landingSpringRampDuration);
    float curvedT = Mathf.Pow(t, Mathf.Max(0.01f, _landingSpringRampCurve));
    float currentMul = Mathf.Lerp(_landingSpringStartMultiplier, 1f, curvedT);
    _jointDriver.SetSpringMultiplier(currentMul);

    if (_landingSpringRampTimer <= 0f)
    {
        _jointDriver.SetSpringMultiplier(1f); // ensure clean finish
    }
}
```

**IMPORTANT — cancel ramp if entering airborne again (wind-up entry or state change):**
In `OnCharacterStateChanged`, when `newState == CharacterStateType.Airborne`:
```csharp
_landingSpringRampTimer = 0f; // cancel any active landing ramp
```
Also in `SetJumpWindUpPose()` (or wherever wind-up is entered): cancel the ramp and snap spring to 1f before applying the wind-up pose. Ensure the ramp doesn't interfere with jump launches.

**Test seam — add this public method:**

{

}
