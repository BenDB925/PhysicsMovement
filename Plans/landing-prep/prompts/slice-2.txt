## YOUR TASK: Slice 2 — Landing Pelvis Tilt Suppression + Counter-Lean

Two targeted changes to BalanceController.cs only.

### Part A — Suppress pelvis tilt spike during landing absorb

In the pelvis tilt computation block (around line 1393), gate on the landing absorb phase:

```csharp
float pelvisTiltTarget = 0f;
if (_landingAbsorbTimer > 0f)
{
    // During landing absorb, the speed-delta detector misreads the landing
    // deceleration as intentional slowing and would add forward lean.
    // Suppress it entirely and reset the smoothed baseline so there's no
    // catch-up spike when the absorb window ends.
    _smoothedForwardSpeed = forwardSpeed;  // reset baseline
    _smoothedPelvisTiltDeg = 0f;          // reset smoothed output
    // pelvisTiltTarget stays 0f
}
else
{
    // Existing pelvis tilt logic (unchanged):
    float forwardSpeed = Vector3.Dot(hipsHorizontalVel, facingXZ.normalized);
    _smoothedForwardSpeed = Mathf.Lerp(_smoothedForwardSpeed, forwardSpeed,
        _pelvisTiltSmoothing * 0.5f * Time.fixedDeltaTime);
    float speedDelta = forwardSpeed - _smoothedForwardSpeed;
    float normalizedDelta = Mathf.Clamp(speedDelta / 1.5f, -1f, 1f);
    pelvisTiltTarget = normalizedDelta * _pelvisTiltMaxDeg * _legAnimator.SmoothedInputMag;
    _smoothedPelvisTiltDeg = Mathf.Lerp(_smoothedPelvisTiltDeg, pelvisTiltTarget,
        _pelvisTiltSmoothing * Time.fixedDeltaTime);
}
```

Note: `forwardSpeed` is currently computed inside the block — make sure the variable is in scope. Restructure the block as needed so the existing logic (unchanged) runs in the else branch. Do not change any existing logic in the else path.

### Part B — Replace forward lean with counter-lean during landing absorb

Add a new serialized field:
```csharp
[SerializeField, Tooltip("Degrees of backward tilt added to the upright target during landing absorption to counteract forward pitch. 0 = no counter-lean (legacy). Replaces _landingAbsorbLeanDeg during landing.")]
private float _landingCounterLeanDeg = 2.0f;
```

In `totalPelvisTilt` assembly (line 1443), replace:
```csharp
+ _landingAbsorbLeanDeg * landingAbsorbBlend
```
with:
```csharp
- _landingCounterLeanDeg * landingAbsorbBlend
```
(Note: subtracted, so it tilts backward — fighting the forward pitch rather than assisting it.)

`_landingAbsorbLeanDeg` stays in the code but is no longer applied to `totalPelvisTilt`. It can be kept as a field in case it's needed elsewhere; just don't apply it in this path.

Also set the prefab override for `_landingCounterLeanDeg` to `2.0` on both `PlayerRagdoll.prefab` and `PlayerRagdoll_Skinned.prefab`.
