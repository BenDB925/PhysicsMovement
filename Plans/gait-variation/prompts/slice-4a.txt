## YOUR TASK: Slice 4A — Idle lateral sway

### What to build

When the character has been idle (SmoothedInputMag < 0.05) for > 0.5s (50 physics frames), apply a gentle lateral perturbation force to the hips Rigidbody at ~0.8Hz using a sine wave. Let the balance controller react naturally — micro-steps and weight shifts emerge from its normal recovery, NOT from scripted animation.

**Physics-first approach:**
- Apply the force via `Rigidbody.AddForce` on the hips Rigidbody (lateral axis, world space)
- Amplitude: small enough to stay inside the balance controller's upright correction deadzone
- Start value: force producing ~0.8cm lateral COM displacement peak-to-peak
- Frequency: ~0.8Hz sine wave (period = 1.25s = 125 physics frames)
- Fade in: lerp perturbation from 0 over the 0.5s entry delay
- Fade out: lerp amplitude to 0 over 0.3s (30 frames) when SmoothedInputMag rises above 0.05

**Implementation location:** `LegAnimator.cs` — add idle sway logic inside `FixedUpdate` (or a dedicated method called from FixedUpdate). LegAnimator already has a reference to the hips Rigidbody.

**Gate:** Bypassed entirely when `_disableOrganicVariation = true`. No sway when fallen.

**Serialized fields to add (on LegAnimator):**
- `_idleSwayForce` float — serialized, default 15f (Newtons). This is the tuning knob.
- `_idleSwayFrequency` float — serialized, default 0.8f (Hz)
- `_idleSwayEnterDelay` float — serialized, default 0.5f (seconds)
- `_idleSwayFadeOutDuration` float — serialized, default 0.3f (seconds)

**Private state to add:**
- `_idleTimer` float — counts up while SmoothedInputMag < 0.05, resets when above
- `_swayPhase` float — tracks sine phase in radians (accumulate: `_swayPhase += 2π * _idleSwayFrequency * Time.fixedDeltaTime`)
- `_swayAmplitudeScale` float — current amplitude scale [0,1], lerps in/out

**Hips Rigidbody reference:** LegAnimator already caches `_hipsRb` (or similar). Check the existing field names before adding a new one.
