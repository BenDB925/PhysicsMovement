## Part B -- Per-stride step angle noise (SECOND commit)

File: Assets/Scripts/Character/LegAnimator.cs

### Step 1 -- Regenerate noise at step trigger

Find where new step targets are computed (around the ComputeSwingTarget calls, lines ~998-1036).
When a new non-Invalid StepTarget is computed for the left leg, regenerate _leftStepAngleNoise.
When a new non-Invalid StepTarget is computed for the right leg, regenerate _rightStepAngleNoise.

Pattern for left leg (mirror for right):
   if (!_disableOrganicVariation && _organicRng != null)
   {
       // Walk bounds: +/-8 deg. Sprint bounds: +/-4 deg. Lerp between based on sprint blend.
       float sprintNorm = _playerMovement != null ? _playerMovement.SprintNormalized : 0f;
       float noiseMag = Mathf.Lerp(8f, 4f, Mathf.Clamp01(sprintNorm));
       _leftStepAngleNoise = (float)(_organicRng.NextDouble() * 2.0 - 1.0) * noiseMag;
   }

### Step 2 -- Apply noise in GetEffectiveStepAngle

GetEffectiveStepAngle is currently leg-agnostic (around line 635-642). You have two options:
  Option A: Add a bool isLeftLeg parameter to GetEffectiveStepAngle and add the noise inside.
  Option B: Apply the noise at the two ComputeSwingTarget call sites after calling GetEffectiveStepAngle.
Use whichever fits the existing code more cleanly. Add a one-line comment saying which option you chose.

The noise addition formula:
  effectiveStepAngle += isLeftLeg ? _leftStepAngleNoise : _rightStepAngleNoise;

### Step 3 -- Clamp final value

The final effectiveStepAngle passed to any swing computation MUST be clamped to [30, 90] degrees.
This is the safe range for the current simulation -- outside this range the character can fall.
  effectiveStepAngle = Mathf.Clamp(effectiveStepAngle, 30f, 90f);

When _disableOrganicVariation is true: skip the noise addition entirely. The clamp can still apply
but the result should be identical to the pre-plan output.

Commit message: "feat(05-1): per-stride step angle noise +/-8 deg walk / +/-4 deg sprint with safe bounds"
