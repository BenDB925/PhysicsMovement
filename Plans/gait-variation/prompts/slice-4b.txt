## YOUR TASK: Slice 4B — Idle vertical bob + micro-adjustment step

### Part 1: Vertical bob

**What:** While idle (same condition as 4A sway: SmoothedInputMag < 0.05, planar/vertical speed low, not fallen), add a very gentle vertical oscillation to the hips height target.

**Implementation approach:**
- Add `SetIdleBobOffset(float offset)` public method to `BalanceController.cs` — stores offset in a new `_idleBobHeightOffset` field
- Apply it inside BalanceController's height maintenance calculation, additive to `_standingHipsHeight + _jumpCrouchHeightOffset` (i.e. same pattern as `_jumpCrouchHeightOffset`)
- Drive the offset from `LegAnimator.cs`: compute `bobOffset = sin(_bobPhase) * amplitude * _swayAmplitudeScale` each FixedUpdate and call `_balance.SetIdleBobOffset(bobOffset)`
- When not idle (or `_disableOrganicVariation`), call `SetIdleBobOffset(0f)` to reset cleanly
- Bob uses a separate phase accumulator `_bobPhase` (so it's independent of sway phase)

**Serialized fields to add (on LegAnimator):**
- `_idleBobAmplitude` float — serialized, default 0.004f (metres, ~0.4cm peak)
- `_idleBobFrequency` float — serialized, default 0.3f (Hz, slow breathing-like)

**Private state to add:**
- `_bobPhase` float — accumulates: `_bobPhase += 2π * _idleBobFrequency * Time.fixedDeltaTime`

**Gate:** Same idle condition as 4A. Bypassed entirely when `_disableOrganicVariation = true`. Must call `SetIdleBobOffset(0f)` when exiting idle to clean up.

### Part 2: Micro-adjustment step

**What:** If the character has been idle for > 2.5s AND has drifted > 1cm from its idle-start position (due to sway), trigger a corrective re-plant on whichever foot is furthest from centre.

**Implementation:**
- Track `_idleStartPosition` (Vector3) — capture hips position when idle starts (when `_idleTimer` first crosses `_idleSwayEnterDelay`)
- Track `_lastMicroStepTime` — to rate-limit micro-steps (no more than one per 1.5s)
- When `_idleTimer > 2.5f` AND `Vector3.Distance(hipsPosition, _idleStartPosition) > 0.01f` AND enough time since last micro-step:
  - Compare left and right foot lateral distance from hips
  - Set `_forcePlantLeft = true` or `_forcePlantRight = true` on whichever foot is furthest
  - Record `_lastMicroStepTime = Time.time`
- The forced plant should use the normal gait step mechanism — just flag that a step is due, let the normal step planting code handle it. Look at how existing stuck-leg recovery forces a step and follow the same pattern.

**Serialized fields to add (on LegAnimator):**
- `_microStepIdleDelay` float — serialized, default 2.5f (seconds before micro-step fires)
- `_microStepDriftThreshold` float — serialized, default 0.01f (metres drift to trigger)
- `_microStepCooldown` float — serialized, default 1.5f (minimum seconds between micro-steps)

**Exposed for tests:**
- `internal int MicroStepCount` — increments each time a micro-step fires (so tests can assert it happened)
