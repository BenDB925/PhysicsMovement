## What this slice does
Adds per-stride lateral foot placement noise. Each time a new step target is computed,
the foot lands slightly wider or narrower than the default. This is independent of the
step angle noise from slice 1 -- it affects WHERE the foot plants, not the swing angle.

The existing StepTarget struct already has a WidthBias field (float, clamped -1..1 by its
constructor). WidthBias is already consumed by StepPlanner and flows through to the final
landing position. This slice perturbs WidthBias at the LegAnimator level, after the planner
has done its job, by storing a per-stride lateral noise value and adding it when the step
target is rebuilt with the modified WidthBias.

## Part A -- Add lateral noise fields and generation (FIRST commit)

File: Assets/Scripts/Character/LegAnimator.cs

### Step 1 -- Add noise fields (near the existing _leftStepAngleNoise / _rightStepAngleNoise)
  private float _leftLateralNoise;
  private float _rightLateralNoise;

### Step 2 -- Reset in SetOrganicVariationSeedForTest
  _leftLateralNoise = 0f;
  _rightLateralNoise = 0f;

### Step 3 -- Generate noise at step trigger
In the same block where _leftStepAngleNoise is generated (when leftStepTarget.IsValid),
also generate _leftLateralNoise. Mirror for right. Use the SAME draw as step angle
(one NextDouble() call per leg per step), but it must be a SEPARATE draw -- do NOT
reuse the step angle draw. The order must be: angle draw first, then lateral draw.
Keep this order fixed and document it with a comment so future slices know which
draw index is which.

  // Draw order per leg per step: [0] step angle noise, [1] lateral noise.
  // Do not reorder -- tests rely on this sequence.
  float latNoiseMag = Mathf.Lerp(0.15f, 0.07f, Mathf.Clamp01(sprintNorm));
  _leftLateralNoise = (float)(_organicRng.NextDouble() * 2.0 - 1.0) * latNoiseMag;

Design: +/-0.15 at walk speed, +/-0.07 at sprint. Sprint is tighter (same principle as
step angle). WidthBias is already clamped to [-1, 1] by the StepTarget constructor so
there is no need to clamp here -- but do NOT pass a value that would push a normal
WidthBias past +/-0.5 to avoid overriding the planner's turn/catch-step logic.
Clamp the final value passed as WidthBias to [-0.5, 0.5] as a safety ceiling.

### Step 4 -- Apply noise when rebuilding the StepTarget
After leftStepTarget is computed (and lateral noise generated), rebuild the target
with the perturbed WidthBias:

  if (leftStepTarget.IsValid && !_disableOrganicVariation)
  {
      float newWidthBias = Mathf.Clamp(leftStepTarget.WidthBias + _leftLateralNoise, -0.5f, 0.5f);
      leftStepTarget = new StepTarget(
          leftStepTarget.LandingPosition,
          leftStepTarget.DesiredTiming,
          newWidthBias,
          leftStepTarget.BrakingBias,
          leftStepTarget.Confidence,
          leftStepTarget.RequestedClearanceHeight);
  }

Mirror for right. Place this rebuild AFTER the noise generation block.
