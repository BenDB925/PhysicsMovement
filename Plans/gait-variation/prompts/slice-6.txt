## YOUR TASK: Slice 6 — Momentum Lean on Direction Change

When the character changes direction (turns), they should lean slightly into the turn — a subtle lateral tilt toward the inside of the curve. This is purely visual (BalanceController visual lean target only — NOT physics forces). It stacks additively with the existing forward/back lean in `totalPelvisTilt`.

---

### Design

**Signal to use:** Angular yaw rate of the character's facing direction. This is cleanly available by tracking the change in facing direction between FixedUpdate frames in BalanceController.

**New private state in BalanceController:**
```csharp
private Vector3 _prevFacingDirection;
private float _smoothedYawRateDeg;   // smoothed yaw rate in deg/s
private float _momentumLeanDeg;      // current smoothed lateral lean contribution
```

**New serialized fields in BalanceController:**
```csharp
[Header("Momentum Lean")]
[SerializeField, Tooltip("Maximum lateral lean angle (degrees) at full turn rate. " +
    "2–3 degrees looks natural without looking exaggerated. Default 2.5.")]
[Range(0f, 8f)]
private float _momentumLeanMaxDeg = 2.5f;

[SerializeField, Tooltip("Yaw rate (deg/s) that produces full lean. " +
    "At MaxFacingTurnRate=540 deg/s this should be around 180–270 for a gentle lean.")]
[Range(30f, 540f)]
private float _momentumLeanFullTurnRate = 200f;

[SerializeField, Tooltip("Smoothing speed for the momentum lean signal (units/s). " +
    "Higher = more responsive. Default 6.")]
[Range(1f, 20f)]
private float _momentumLeanSmoothing = 6f;
```

**Yaw rate computation (in FixedUpdate, before STEP 3.9 or just before STEP 4):**

```csharp
// Track facing direction change to derive yaw rate
Vector3 currentFacing = Vector3.ProjectOnPlane(
    _currentBodySupportCommand.FacingDirection, Vector3.up);

float instantYawRate = 0f;
if (currentFacing.sqrMagnitude > 0.001f && _prevFacingDirection.sqrMagnitude > 0.001f
    && Time.fixedDeltaTime > 0f)
{
    float angleDelta = Vector3.SignedAngle(
        _prevFacingDirection.normalized,
        currentFacing.normalized,
        Vector3.up);
    instantYawRate = angleDelta / Time.fixedDeltaTime;
}
_prevFacingDirection = currentFacing.sqrMagnitude > 0.001f ? currentFacing.normalized : _prevFacingDirection;

_smoothedYawRateDeg = Mathf.Lerp(_smoothedYawRateDeg, instantYawRate,
    Time.fixedDeltaTime * _momentumLeanSmoothing);
```

**Lean computation:**

```csharp
float momentumLeanContribution = 0f;
if (_momentumLeanMaxDeg > 0f && !IsFallen && effectivelyGrounded)
{
    float normalizedYaw = Mathf.Clamp(_smoothedYawRateDeg / _momentumLeanFullTurnRate, -1f, 1f);
    // Positive yaw = turning left (counter-clockwise) → lean left (negative lateral = left in facing space)
    momentumLeanContribution = normalizedYaw * _momentumLeanMaxDeg * _legAnimator.SmoothedInputMag;
}
```

**Inject into totalPelvisTilt as a LATERAL lean (not forward/back):**

The existing `totalPelvisTilt` is a forward/back pitch around the lateral axis. Momentum lean is a roll around the forward axis (lean sideways into the turn). These are different axes and should NOT be summed into the same scalar.

Instead, modify the uprightTarget construction (around line 1470 in BalanceController) to apply the lateral lean separately:

```csharp
// Existing forward/back tilt:
Vector3 uprightTarget = Vector3.up;
if (Mathf.Abs(totalPelvisTilt) > 0.01f)
{
    Vector3 tiltFacing = Vector3.ProjectOnPlane(
        _currentBodySupportCommand.FacingDirection, Vector3.up);
    if (tiltFacing.sqrMagnitude > 0.001f)
    {
        Vector3 lateralAxis = Vector3.Cross(Vector3.up, tiltFacing.normalized);
        uprightTarget = Quaternion.AngleAxis(totalPelvisTilt, lateralAxis) * Vector3.up;
    }
}

// NEW: lateral momentum lean — rotate uprightTarget sideways into the turn
if (Mathf.Abs(momentumLeanContribution) > 0.01f)
{
    Vector3 facingXZ = Vector3.ProjectOnPlane(
        _currentBodySupportCommand.FacingDirection, Vector3.up);
    if (facingXZ.sqrMagnitude > 0.001f)
    {
        // Lean around the facing axis (forward axis) — positive = lean left
        uprightTarget = Quaternion.AngleAxis(-momentumLeanContribution, facingXZ.normalized) * uprightTarget;
    }
}
```

**Gates:**
- Only when `effectivelyGrounded && !IsFallen`
- `_legAnimator.SmoothedInputMag` scales it to zero at idle
- When `_suppressPelvisExpression` is true, also zero this lean
- When `_landingAbsorbTimer > 0f`, also zero this lean (same suppression as tilt)

**Initialisation:** In `Start` or `OnEnable`, set `_prevFacingDirection = Vector3.forward` as a safe default.
