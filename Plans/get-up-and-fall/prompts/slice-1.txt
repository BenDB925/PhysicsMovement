## HANDOFF CONTEXT

ProceduralStandUp.cs already exists (435 lines) and is wired up in CharacterState and the prefab. The 4-phase get-up sequence runs, but the character launches into the air after getting up. Root cause: `_forcedStandImpulse = 350f` fires as a ForceMode.Impulse after `_maxStandUpAttempts` is exhausted — 350 N·s is twice the jump force. This slice adds debug visibility and fixes the rocket launch. It does NOT tune the phase values — that's slice 2c after Benny plays with the debug log.

### Change 2: Fix the rocket launch

**Replace `ApplyForcedStand()`** with a multi-frame force version.

Add new serialized fields:
```csharp
[SerializeField, Range(1, 30)] private int _forcedStandFrames = 8;
```

Change `_forcedStandImpulse` default from `350f` to `60f`.

Add a private counter field:
```csharp
private int _forcedStandFrameCounter;
```

In `ApplyForcedStand()`, instead of `_hipsRb.AddForce(Vector3.up * _forcedStandImpulse, ForceMode.Impulse)`, set:
```csharp
_forcedStandFrameCounter = _forcedStandFrames;
```

Then in `FixedUpdate()`, after the phase switch, add:
```csharp
if (_forcedStandFrameCounter > 0)
{
    if (_hipsRb != null)
        _hipsRb.AddForce(Vector3.up * _forcedStandImpulse, ForceMode.Force);
    _forcedStandFrameCounter--;
}
```

This spreads the force over `_forcedStandFrames` frames (default 8 = 0.08s at 100Hz) instead of a single impulse pop.

The rest of `ApplyForcedStand()` stays as-is (ClearSurrender, ResetSpringProfile, reset counters, fire OnCompleted).

### Change 3: Fix _groundY drift

In `FixedUpdate()`, at the top before the phase switch, add:
```csharp
if (IsActive && _ragdollSetup != null && _hipsRb != null)
{
    foreach (Rigidbody rb in _ragdollSetup.AllBodies)
    {
        _groundY = Mathf.Min(_groundY, rb.position.y);
    }
}
```

`_groundY` must only ever decrease (use `Mathf.Min`) — never raise it. A limb lifting up during get-up must not push `_groundY` upward and make height gates easier to pass.
