## HANDOFF CONTEXT

Slices 1 and 2a are merged. The character now crumples naturally on surrender (joints soften over 0.25s), and the forced-stand rocket launch is fixed. The 0.4s GettingUp minimum dwell is also in.

Current problem with `TickOrientProne` in ProceduralStandUp.cs:

```csharp
Vector3 currentForward = _hipsRb.transform.forward;
Vector3 desiredForward = Vector3.down;
Vector3 rollAxis = Vector3.Cross(currentForward, desiredForward).normalized;
if (rollAxis.sqrMagnitude > 0.001f)
{
    _hipsRb.AddTorque(rollAxis * _orientTorque, ForceMode.Force);
}
```

This tries to rotate `hips.forward` toward `Vector3.down`. But:
1. It does not check whether the character is already face-down or face-up — it applies the same torque regardless
2. It does not check whether the torque is being applied in the correct direction for the current pose
3. When the character is face-up (lying on their back), the torque may roll them the wrong way or oscillate
4. The `_proneDotThreshold` check uses `Dot(hips.forward, down)` but this advances even if the character rolled onto their back instead of their front

The correct approach:
- `Dot(hips.up, Vector3.up) > threshold` means face-up (lying on back)
- `Dot(hips.up, Vector3.up) < -threshold` means face-down (lying on front) — this is what we want
- Torque direction should be chosen to roll toward face-down via the shorter arc

Replace the entire roll torque block in `TickOrientProne` with correct face-up-aware logic:

**New approach:**
- Desired pose: hips.up points toward `Vector3.down` (face-down/prone)
- Compute the torque axis as `Cross(hipsUp, Vector3.down)` — this gives the shortest-arc rotation from current hips.up toward world-down
- This automatically handles both face-up and face-down cases and always rolls toward prone

```csharp
// Roll hips so the up-vector points toward world-down (prone/face-down).
// Cross(hipsUp, targetUp) gives the shortest-arc rotation axis regardless of
// whether the character is currently face-up or face-down.

Vector3 desiredUp = Vector3.down;
Vector3 rollAxis = Vector3.Cross(hipsUp, desiredUp).normalized;
if (rollAxis.sqrMagnitude > 0.001f)
{
    _hipsRb.AddTorque(rollAxis * _orientTorque, ForceMode.Force);
}
```

The existing advance condition uses:
