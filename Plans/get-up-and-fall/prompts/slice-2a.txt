## HANDOFF CONTEXT

Slice 1 is complete. Debug logging is in ProceduralStandUp.cs, and the forced-stand rocket launch is fixed (60N over 8 frames instead of 350N impulse).

The remaining problem: when the character falls over, it doesn't look like a natural collapse. The arms stay rigid and pinned to the body, the character braces instead of going limp. The body tumbles stiffly.

Root cause (confirmed by reading the code):
- `TriggerSurrender()` in BalanceController.cs sets `_ragdollSetup.SetSpringProfile(0.25f, 0.25f, 0.25f, 0.15f)`
- The `0.15f` blend duration is nearly instant — spring profile snaps to 25% stiffness rather than ramping
- `0.25f` multiplier may also be too high — arms at 25% stiffness still feel rigid enough to brace
- The support scales (UprightStrengthScale, HeightMaintenanceScale, StabilizationScale) also snap to 0f immediately — no ramp

This slice replaces the snap with a proper crumple ramp so the body softens naturally over ~0.2s.

### Change 2: Replace snap with ramp in TriggerSurrender

Find the `TriggerSurrender` method. It currently has:
```csharp
CancelAllRamps();
IsSurrendered = true;
UprightStrengthScale = 0f;
HeightMaintenanceScale = 0f;

```

Replace the three immediate zero-sets with ramp calls. The ramps must be set up AFTER `CancelAllRamps()` (which clears any in-progress ramps). Change to:

```csharp
CancelAllRamps();

RampUprightStrength(0f, _surrenderCrumpleDuration);
RampHeightMaintenance(0f, _surrenderCrumpleDuration);
RampStabilization(0f, _surrenderCrumpleDuration);
```

### Change 3: Lower spring multipliers and add blend duration

Find the `SetSpringProfile` call inside `TriggerSurrender`:
```csharp
_ragdollSetup.SetSpringProfile(0.25f, 0.25f, 0.25f, 0.15f);
```

Replace with:
```csharp
_ragdollSetup.SetSpringProfile(0.05f, 0.08f, 0.05f, _surrenderCrumpleDuration);
```

Values rationale:
- Arms (0.05f): nearly limp — arms should flop and hit the ground, not brace
- Legs (0.08f): slightly stiffer than arms — legs naturally have more tension even when falling
- Torso (0.05f): limp — should crumple under momentum
- Blend duration: use `_surrenderCrumpleDuration` so the spring ramp matches the support scale ramp

### Change 4: Add ImpactKnockdown test seam

To support future tests that trigger falls via physics impact, add a public method to BalanceController that applies a force impulse to the hips Rigidbody and then immediately triggers surrender. This is a test seam — gated so it can only be called in tests.

Find where `_hipsRb` or the main Rigidbody is referenced in BalanceController. Add the following method:

```csharp
/// <summary>
/// Test seam: applies a horizontal impulse to the hips and triggers surrender.
/// Simulates an external impact knockdown for use in PlayMode tests only.
/// </summary>
/// <param name="direction">World-space direction of the impulse (will be normalized).</param>
/// <param name="force">Impulse magnitude in Newtons.</param>
public void TriggerImpactKnockdownForTest(Vector3 direction, float force)
{
    Rigidbody rb = GetComponent<Rigidbody>();
    if (rb != null)
    {
        rb.AddForce(direction.normalized * force, ForceMode.Impulse);
    }
    TriggerSurrender(1f);
}
```
