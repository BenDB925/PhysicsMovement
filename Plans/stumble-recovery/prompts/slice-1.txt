# Plan 11 Slice 1 — Passive Tilt Classification in LocomotionDirector

## Context

The catch-step and arm-brace pipeline already exists:
- `LocomotionDirector.ClassifyRecoverySituation` → `RecoverySituation.Stumble`
- `LegAnimator` → `LegStateType.CatchStep`
- `ArmAnimator` → brace pose via `IsRecoveryActive`

The gap: classification only fires on locomotion collapse (walking + not progressing + intent).
A physics hit while standing or walking never triggers it.

Confirmed from editor test: tilt went 0.2° → 66.8° (Fallen) in a handful of frames with
RecoverySituation.None the whole time.

## Goal for this slice

Add two new classification paths to `LocomotionDirector.ClassifyRecoverySituation` so
passive destabilisation (hit, push, collision) triggers `RecoverySituation.Stumble`:

1. **Angular velocity spike** — fast impact rotates hips rapidly → fire immediately,
   bypass debounce
2. **Sustained tilt** — hips above 20° for normal debounce window → fire via normal path

That's it. One file changed. No step planner changes in this slice.

### Sustained tilt path — add to `ClassifyRecoverySituation` BEFORE the `IsLocomotionCollapsed` check:

Gates:
- NOT intentional jump sequence (`isIntentionalJumpSequence` already computed in that method)
- NOT `_balanceController.IsSurrendered`
- NOT `_balanceController.IsFallen`
- `_balanceController.IsGrounded`
- `_characterState.CurrentState` is `Standing` or `Moving`
  (cache `_characterState` in `Awake` — check if it's already cached; if not, add it)
- `_balanceController.UprightAngle >= _passiveTiltAngleTrigger`

When all gates pass: `return RecoverySituation.Stumble;`

This goes through the normal `_transitionGuard` debounce — fine for slow pushes.
